Starting & Ending a Session
with Codex or Claude

The two-part ritual — and the one mistake that quietly loses your work.

First, the key idea

Two separate things

The state log

A journal

A markdown summary of where the project stands. Human + AI readable.
The codebase

The real source of truth

Your repo, pushed to GitHub with git.

“log this” summarizes the work — it does not push your code.

Starting a session

Tell the assistant where you left off

/init state

It reads the state log and recaps the last session.

Ending a session

Two parts — in this order

1 · In your terminal
git add -A && git commit -m "…" && git push
This actually saves your code to GitHub.
2 · To the assistant
log this
This records what you did in the journal.
The honest catch

“log this” does not push your code

Log only, and the journal will say the code is saved while it sits unpushed on your Mac. Safer:

check git status, then log this

Now the assistant verifies the repo first, then logs reality.

Best practice

Keep the log in the repo

scripts/STATE_LOG.md

Both Codex and Claude can read & edit it, and git tracks every change. Mirror to Drive only when you want it on mobile / NotebookLM.

Remember

Two parts, every session:
push your code, then log it.
